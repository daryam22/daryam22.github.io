First, the id of this form was made into "signup," so any CSS applied to this form would be under "form#signup." All the headings were styled under "formA#signup h2,h3,h4" and the border surrounding the Delivery options, choose format, choose frequency, and comments is under "form#signup fieldset." Additionally, the legend in the fieldset enables the text "Delivery Options" to appear on the border. The "form#signup text area" pertains to css applied to the Comments text box and the "form#signup text area:focus" makes Comments box turn white when a person clicks inside it. The type of id for the "Categories of Interest" was set as a "checkbox" in the html and the type of id for the "Choose the format" was set as a "radio" in the html. Therefore, "form#signup input[type="checkbox"], form#signup input[type="radio"]" set the width, padding, margin, and opacity of the checkbox and radio that show up when a user clicks on any options in "Categories of Interest" and "Choose the format." "form#signup input#email" applies CSS to the email box in this form; it altered font-size, background-color, etc... Additionally, "form#signup input#email:hover" enables the user to hover over the box and it will turn white. Lastly, the "form#signup input[type="submit"]" sets the CSS for the submit button; it sets the background-color, font-weight, etc... and "form#signup input[type="submit"]:hover" enables the user to hover over the submit button and make it turn the color of #ffd34c.
